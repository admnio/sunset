# Sunset v1.3.0 — Queue Pause/Resume Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Tasks use checkbox (`- [ ]`) syntax.

**Goal:** Per-queue pause/resume across SQS, Redis, and RabbitMQ transports. Soft signal — checked at pop() time. Dashboard buttons + artisan commands + public contract for programmatic use.

**Architecture:** Redis SET `sunset:queues:paused` of `"{connection}:{queue}"` members. `QueuePauseGate` (fail-open on Redis down) is called at the top of each transport's `pop()`, returning null when paused. Two new public events (`QueuePaused`, `QueueResumed`) integrate with the activity stream.

**Spec:** `docs/superpowers/specs/2026-05-21-v1.3.0-queue-pause-resume-design.md`.

### Task 1: Contract + Events + Repository

**Files:**

- Create `src/Events/QueuePaused.php` — public, NO `@internal`. Properties: `public readonly string $connection`, `public readonly string $queue`, `public readonly ?string $actor`.
- Create `src/Events/QueueResumed.php` — same shape.
- Create `src/Repositories/Redis/RedisQueuePauseRepository.php` — `@internal`. Implements contract directly (NO separate write method like other repos — write methods ARE on the contract per spec).
- Tests:
  - `tests/Unit/QueuePause/QueuePausedEventTest.php` — constructor + property exposure (small, but locks the public shape)
  - `tests/Unit/QueuePause/QueueResumedEventTest.php`
  - `tests/Integration/QueuePause/RedisQueuePauseRepositoryTest.php` — against real Redis. Cover: pause/resume/isPaused/all in all combinations, idempotence, cross-connection isolation, `all()` returns `list<array{connection, queue}>` not `list<string>`.
  - `tests/Unit/QueuePause/RedisQueuePauseRepositoryFailureTest.php` — mock RedisFactory throws → reads degrade to `[]`/false, writes re-throw.

Contract shape:
```php
public function pause(string $connection, string $queue): void;
public function resume(string $connection, string $queue): void;
public function isPaused(string $connection, string $queue): bool;
public function all(): array;  // list<array{connection: string, queue: string}>
```

`pause()` and `resume()` MUST fire the corresponding event after successful SADD/SREM. The repository constructor takes `RedisFactory` AND `Illuminate\Contracts\Events\Dispatcher` so it can dispatch. Decision: the repository fires the events, not the controller/command. This way programmatic API consumers also get the events automatically.

`actor` is an OPTIONAL second parameter on both methods:
```php
public function pause(string $connection, string $queue, ?string $actor = null): void;
public function resume(string $connection, string $queue, ?string $actor = null): void;
```

Forward `$actor` into the dispatched event.

### Task 2: QueuePauseGate

**Files:**
- Create `src/QueuePause/QueuePauseGate.php` — `@internal`
- Create `tests/Unit/QueuePause/QueuePauseGateTest.php`

Constructor: `QueuePauseRepository $repository`, `LoggerInterface $logger`, `Closure $clock` (for the log-throttle), optional `int $warnIntervalSeconds = 60`.

Single method: `isPaused(string $connection, string $queue): bool`. Wraps `$repository->isPaused()` in try/catch. On throw: fail open (return `false`), log warning at most once per `warnIntervalSeconds` using an internal `lastWarnAt` float.

Tests:
- Delegates to repository when no exception → returns what repo returns.
- Repository throws → returns false.
- Repository throws repeatedly → only one warning log within 60s window (use injected clock).
- After 60s, throttle window resets and a second warning fires.

### Task 3: Transport pop() integration

**Files modified:**
- `src/Transports/Sqs/SqsQueue.php` — add gate check at top of `pop()`
- `src/Transports/Redis/RedisQueue.php` — add gate check at top of `pop()`
- `src/Transports/Rabbit/RabbitQueue.php` — add gate check at top of `pop()`

**Tests created:**
- `tests/Integration/QueuePause/SqsQueuePauseTest.php`
- `tests/Integration/QueuePause/RedisQueuePauseTest.php`
- `tests/Integration/QueuePause/RabbitQueuePauseTest.php`

Pattern in each transport (mirror existing rate-limit gate pattern):

```php
public function pop($queue = null)
{
    $queueName = $queue ?: $this->default;

    if ($this->container->make(\Admnio\Sunset\QueuePause\QueuePauseGate::class)
            ->isPaused($this->getConnectionName(), $queueName)) {
        return null;
    }

    // ...existing pop() body
}
```

For SqsQueue, the `$queue` param normalization is different — examine the existing code (it uses `$this->getQueue($queue)` to translate to URLs). Use the bare queue name for the pause check, NOT the URL.

For RedisQueue/RabbitQueue, `$queue ?: $this->default` is the right name.

Tests: enqueue a job (or stub the underlying client), call `$queue->pop('some-queue')`, expect null when paused. When NOT paused, expect the underlying parent::pop() to be reached (verify via existing pop-success integration tests still passing).
