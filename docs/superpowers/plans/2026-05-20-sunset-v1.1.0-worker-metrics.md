# Sunset v1.1.0 — Worker Metrics Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship per-worker CPU and memory monitoring to the Sunset dashboard.

**Architecture:** Hook Laravel's `Looping` queue event, throttle samples to once per 5s, write per-PID hash + sorted-set series to Redis. Read on the Supervisors page through a new public `WorkerMetricsRepository` contract.

**Tech Stack:** PHP 8.2+ / Laravel 10–12, phpredis or predis via `Illuminate\Contracts\Redis\Factory`, Inertia + Vue 3 + Tailwind on the dashboard.

**Spec:** `docs/superpowers/specs/2026-05-20-v1.1.0-worker-metrics-design.md` — read it first if you need context.

### Task 3: Sampler service (pure)

**Files:**
- Create: `src/Telemetry/WorkerMetricsSampler.php`
- Create: `tests/Unit/Telemetry/WorkerMetricsSamplerTest.php`

The sampler is a pure, deterministic class that owns the throttling and CPU-delta math. It takes a clock callable (returns float wall-time) and a resource-usage callable (returns array with user_sec/sys_sec/wall_sec/rss_bytes/pid). No Laravel deps.

- [ ] **Step 1:** Write `WorkerMetricsSamplerTest`. Cover:
  - First call returns a snapshot with `cpu_pct == null`.
  - Second call before `interval_seconds` elapsed returns `null` (throttled).
  - Second call after `interval_seconds` returns a snapshot with `cpu_pct == ((u2+s2)-(u1+s1))/(wall2-wall1)*100`.
  - On Windows-like input (cpu_delta == 0 across two samples), `cpu_pct` remains `null`.
  - `jobsProcessed` counter increments per `recordJob()` call.
  - `startedAt` is captured on the first sample and preserved.

- [ ] **Step 2:** Run the test, confirm it fails.

- [ ] **Step 3:** Implement the sampler:

```php
namespace Admnio\Sunset\Telemetry;

class WorkerMetricsSampler
{
    private ?float $lastWall = null;
    private ?float $lastCpuSeconds = null;
    private int $startedAt = 0;
    private int $jobsProcessed = 0;
    private int $consecutiveZeroCpuSamples = 0;

    public function __construct(
        private int $intervalSeconds,
        private \Closure $clock,                   // fn(): float — unix wall seconds
        private \Closure $resourceUsage            // fn(): array{user_sec: float, sys_sec: float, rss_bytes: int, pid: int}
    ) {}

    public function recordJob(): void { $this->jobsProcessed++; }

    public function sample(string $supervisor = null, string $connection = null, array $queues = null): ?WorkerMetricsSnapshot
    {
        $now = ($this->clock)();
        if ($this->lastWall !== null && ($now - $this->lastWall) < $this->intervalSeconds) {
            return null;
        }

        $usage = ($this->resourceUsage)();
        $cpuSeconds = $usage['user_sec'] + $usage['sys_sec'];
        $cpuPct = null;

        if ($this->lastWall !== null && $this->lastCpuSeconds !== null) {
            $cpuDelta = $cpuSeconds - $this->lastCpuSeconds;
            $wallDelta = max($now - $this->lastWall, 0.001);
            if ($cpuDelta === 0.0) {
                $this->consecutiveZeroCpuSamples++;
                $cpuPct = $this->consecutiveZeroCpuSamples >= 2 ? null : 0.0;
            } else {
                $this->consecutiveZeroCpuSamples = 0;
                $cpuPct = ($cpuDelta / $wallDelta) * 100;
            }
        }

        if ($this->startedAt === 0) {
            $this->startedAt = (int) $now;
        }

        $snapshot = new WorkerMetricsSnapshot(
            pid: $usage['pid'],
            supervisor: $supervisor,
            connection: $connection,
            queues: $queues,
            startedAt: $this->startedAt,
            rssBytes: $usage['rss_bytes'],
            cpuPct: $cpuPct,
            jobsProcessed: $this->jobsProcessed,
            lastReportAt: (int) $now,
        );

        $this->lastWall = $now;
        $this->lastCpuSeconds = $cpuSeconds;

        return $snapshot;
    }
}
```

  - `find()` returns null for unknown PID, returns a `WorkerMetricsSnapshot` for a recorded PID.
  - `all()` returns one snapshot per recorded PID, skips PIDs whose hash expired (manual flushExpired analog or set TTL trick — see Task 6).
  - `series('rss')` returns up to `series_points` newest entries in ascending time order.
  - Series sorted set is capped to `series_points` entries after repeated `record()` calls.
  - Hash TTL is set to 30 seconds on each write.
  - Reads return `[]` / null gracefully when Redis throws (use a mock factory for this case in a separate unit test).

- [ ] **Step 3:** Implement the repository. Use the existing `key()` + `connection()` private-method pattern from `RedisProcessRepository`. Read `sunset.telemetry.series_points` from config for the cap.

```php
public function record(WorkerMetricsSnapshot $snapshot): void
{
    $pid = $snapshot->pid;
    $hashKey = $this->key("worker_metrics:{$pid}");
    $rssKey  = $this->key("worker_metrics:{$pid}:series:rss");
    $cpuKey  = $this->key("worker_metrics:{$pid}:series:cpu");
    $setKey  = $this->key('worker_metrics:pids');
    $cap     = (int) config('sunset.telemetry.series_points', 60);

    $this->connection()->pipeline(function ($pipe) use ($snapshot, $hashKey, $rssKey, $cpuKey, $setKey, $cap) {
        $pipe->hmset($hashKey, $snapshot->toArray() + [
            'queues' => json_encode($snapshot->queues ?? []),
        ]);
        $pipe->expire($hashKey, 30);
        $pipe->sadd($setKey, $snapshot->pid);

        $pipe->zadd($rssKey, $snapshot->lastReportAt, "{$snapshot->lastReportAt}:{$snapshot->rssBytes}");
        $pipe->zremrangebyrank($rssKey, 0, -1 * ($cap + 1));
        $pipe->expire($rssKey, 600);

        if ($snapshot->cpuPct !== null) {
            $pipe->zadd($cpuKey, $snapshot->lastReportAt, "{$snapshot->lastReportAt}:" . (int) round($snapshot->cpuPct * 100));
            $pipe->zremrangebyrank($cpuKey, 0, -1 * ($cap + 1));
            $pipe->expire($cpuKey, 600);
        }
    });
}
```

For the read side (`all`, `find`, `series`):

- `all()` SMEMBERS the pids set, pipelines HGETALL for each, filters out empty results (TTL'd), reconciles the pids set by SREM-ing dead pids, and returns snapshots.
- `find($pid)` HGETALL the hash, return null on empty.
- `series($pid, 'rss'|'cpu', $max)` ZRANGE the corresponding sorted set with `withscores=true`. Members are stored as `"{ts}:{value}"` so the value is parseable; parse and return `[{ts: int, value: int|float}, ...]`. Divide CPU by 100 to restore the original float.

When parsing the hash, JSON-decode `queues`, cast types appropriately, hand to `WorkerMetricsSnapshot::fromArray()`.
