# Sunset v0.3.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `RedisTransport` so Redis-backed Laravel queues get the same Sunset/Horizon dashboard parity SQS users got in v0.2.0.

**Architecture:** Mirror v0.2.0's SQS pattern. Extend `Illuminate\Queue\RedisQueue` and dispatch Horizon's `JobPending`/`JobPushed`/`JobReserved` events ourselves. Implement the `Admnio\Sunset\Contracts\Transport` interface (registered in `TransportRegistry` alongside `SqsTransport`). Override Laravel's stock `'redis'` driver via load-order priority. Delegate all connection management to `Illuminate\Contracts\Redis\Factory` — cluster/sentinel/predis-vs-phpredis stay in `config/database.php`.

**Tech Stack:** PHP 8.2+, Laravel 10/11/12, Horizon 5.x, PHPUnit 11, Orchestra Testbench, Redis 7 (already in docker-compose).

**Reference spec:** `docs/superpowers/specs/2026-05-18-sunset-v0.3.0-redis-transport.md`

**Working directory:** `C:\WebDev\Packages\HorizonSQS`

- Docker: Bash `docker compose up -d` (LocalStack + Redis); LocalStack ignored for v0.3.0 tests but Redis required for integration

### Step 2.3: Implement `RedisTransport`

Create `src/Transports/Redis/RedisTransport.php`:

```php
<?php

namespace Admnio\Sunset\Transports\Redis;

use Admnio\Sunset\Contracts\Transport;
use Illuminate\Contracts\Queue\Queue;
use Illuminate\Contracts\Redis\Factory as RedisFactory;
use Psr\Log\LoggerInterface;
use Psr\Log\NullLogger;
use Throwable;

class RedisTransport implements Transport
{
    public function __construct(
        private RedisFactory $redis,
        private array $packageConfig,
        private ?LoggerInterface $logger = null,
    ) {
    }

    public function name(): string
    {
        return 'redis';
    }

    public function connect(array $config): Queue
    {
        return new RedisQueue(
            $this->redis,
            $config['queue'] ?? 'default',
            $config['connection'] ?? 'default',
            $config['retry_after'] ?? 60,
            $config['block_for'] ?? null,
        );
    }

    public function workload(array $queues): array
    {
        $logger = $this->logger ?? new NullLogger();
        $connectionName = $this->packageConfig['transports']['redis']['workload_connection'] ?? 'default';
        $conn = $this->redis->connection($connectionName);

        $records = [];
        foreach ($queues as $queue) {
            try {
                $length = (int) $conn->llen("queues:{$queue}");
                $length += (int) $conn->zcard("queues:{$queue}:delayed");
                $length += (int) $conn->zcard("queues:{$queue}:reserved");
            } catch (Throwable $e) {
                $logger->warning('sunset: Redis workload query failed for queue', [
                    'queue' => $queue,
                    'error' => $e->getMessage(),
                ]);
                $length = 0;
            }

            $records[] = [
                'name' => $queue,
                'length' => $length,
                'wait' => 0,
                'processes' => 0,
                'split_queues' => null,
            ];
        }

        return $records;
    }
}
```

**Note on constructor args for `RedisQueue`:** Laravel's `Illuminate\Queue\RedisQueue` accepts `($redis, $default, $connection, $retryAfter, $blockFor)` across Laravel 10/11/12. Newer Laravel adds `$dispatchAfterCommit` and `$migrationBatchSize` with default values — those defaults are fine; we don't pass them. If a future Laravel version makes one of them required, this task's tests will catch it.

### Step 3.3: Implement `RedisQueue`

Create `src/Transports/Redis/RedisQueue.php`:

```php
<?php

namespace Admnio\Sunset\Transports\Redis;

use Illuminate\Queue\RedisQueue as LaravelRedisQueue;
use Illuminate\Support\Str;
use Laravel\Horizon\Events\JobPending;
use Laravel\Horizon\Events\JobPushed;
use Laravel\Horizon\Events\JobReserved;
use Laravel\Horizon\JobPayload;

class RedisQueue extends LaravelRedisQueue
{
    private mixed $lastPushed = null;

    public function push($job, $data = '', $queue = null)
    {
        $this->lastPushed = $job;
        return parent::push($job, $data, $queue);
    }

    public function pushRaw($payload, $queue = null, array $options = [])
    {
        $prepared = (new JobPayload($payload))->prepare($this->lastPushed);

        $this->raiseHorizonEvent(new JobPending($prepared->value), $queue);

        $result = parent::pushRaw($prepared->value, $queue, $options);

        $this->raiseHorizonEvent(new JobPushed($prepared->value), $queue);

        return $prepared->id();
    }

    public function later($delay, $job, $data = '', $queue = null)
    {
        $this->lastPushed = $job;
        return parent::later($delay, $job, $data, $queue);
    }

    public function pop($queue = null)
    {
        $job = parent::pop($queue);

        if ($job !== null) {
            $this->raiseHorizonEvent(new JobReserved($job->getReservedJob()), $queue);
        }

        return $job;
    }

    private function raiseHorizonEvent($event, ?string $queue): void
    {
        $queueName = Str::replaceFirst('queues:', '', $queue ?? $this->default);
        event($event->connection($this->getConnectionName())->queue($queueName));
    }
}
```

**Notes on parent class assumptions:**

- `Illuminate\Queue\RedisQueue` has `protected $default` (the default queue name) and exposes `getConnectionName()` from its `Illuminate\Queue\Queue` parent. Both are stable across Laravel 10/11/12.
- `parent::pop($queue)` returns a `RedisJob` (or `null`). `RedisJob::getReservedJob()` returns the raw reserved JSON payload — exactly what `JobReserved`'s constructor expects.
- The `$payload` arg to `pushRaw` is already a JSON string by the time Laravel's `push()` calls into it; we wrap it via `JobPayload::__construct(string $value)`.

```php
        $this->assertSame(
            'hello-from-redis',
            file_get_contents(sys_get_temp_dir() . '/sunset-marker')
        );
    }

    public function test_dashboard_pending_endpoint_lists_redis_job(): void
    {
        // Push a job; do NOT pop it — leave it pending.
        Queue::push(new RecordingJob('pending-test'));

        // Give Horizon's event listeners a moment to record into Redis.
        usleep(200_000);

        $response = $this->get('/horizon/api/jobs/pending');
        $response->assertOk();

        $data = $response->json();
        $this->assertNotEmpty($data['jobs'] ?? [], 'Expected /horizon/api/jobs/pending to include the redis-driven job');
    }
}
```

**Notes on the test:**
- We override `queue.default` to `'redis'` and use a unique queue name to avoid colliding with other tests' state.
- `RecordingJob` already exists (from v0.1.0 integration tests) and writes the marker. We're reusing the same fixture but updated its marker name in v0.2.0 to `sunset-marker`.
- The dashboard test depends on Horizon's `StoreJob` listener picking up the `JobPushed` event our queue dispatches. With Sunset's `RedisQueue` correctly dispatching the event, the listener runs and the job appears in Horizon's `JobRepository` recent/pending lists.

# Upgrading from `admnio/sunset` v0.2.0 to v0.3.0
