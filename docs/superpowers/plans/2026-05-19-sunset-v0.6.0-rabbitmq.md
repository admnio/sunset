# Sunset v0.6.0 — RabbitMQ Transport

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> **Spec status:** This plan was extracted from an alternate-design exploration that did not produce a separate spec file. The design context is captured in "Design summary" below and inline in each task. A formal companion spec can be written if the design needs revisiting; for now, treat this plan as self-contained.

**Goal:** Add `RabbitTransport` as Sunset's third first-class transport, alongside SQS (v0.2.0) and Redis (v0.3.0). Implements the existing `Transport` contract. Fires the same Sunset lifecycle events at the same points so dashboard observability works uniformly. Reuses the existing `Admnio\Sunset\Delay\DelayedJobStore` for delayed dispatches so delay semantics stay consistent across all three transports.

**Architecture:** Mirror the v0.3.0 Redis pattern. Extend `VladimirYuldashev\LaravelQueueRabbitMQ\Queue\RabbitMQQueue` and override `push`/`pushRaw`/`pop`/`later` to fire `JobQueueing`/`JobQueued`/`JobReserved` events and to route delays through `Admnio\Sunset\Delay\DelayedJobStore`. Implement `Admnio\Sunset\Contracts\Transport` in `RabbitTransport`, register it in `TransportRegistry`. Override Laravel's stock `'rabbitmq'` driver via load-order priority on `QueueManager::addConnector()`.

**Design summary:**
- AMQP library: `vladimir-yuldashev/laravel-queue-rabbitmq` (de-facto Laravel AMQP package).
- Delayed dispatch: reuse the existing Sunset Redis-backed `DelayedJobStore` instead of RabbitMQ's delayed-exchange plugin. Identical delay semantics across SQS, Redis, and RabbitMQ.
- Extended-payload / S3 spillover: out of scope. RabbitMQ handles ~128MB natively.
- Topology: direct exchange, one queue per Laravel queue name. DLX is opt-in via `sunset.transports.rabbitmq.dead_letter.enabled`.
- Workload depth: AMQP `queue.declare(passive: true)` returns ready+unacked counts.

**Tech Stack:** PHP 8.2+, Laravel 10/11/12, PHPUnit 11, Orchestra Testbench, RabbitMQ 3.x (via docker-compose), `vladimir-yuldashev/laravel-queue-rabbitmq ^14.0`, `php-amqplib/php-amqplib`.

```php
<?php

namespace Admnio\Sunset\Transports\Rabbit;

use Admnio\Sunset\Contracts\Transport;
use Illuminate\Contracts\Container\Container;
use Illuminate\Contracts\Queue\Queue;
use PhpAmqpLib\Connection\AMQPStreamConnection;
use Psr\Log\LoggerInterface;
use Psr\Log\NullLogger;
use Throwable;
use VladimirYuldashev\LaravelQueueRabbitMQ\Queue\Connectors\RabbitMQConnector as VendorConnector;

class RabbitTransport implements Transport
{
    public function __construct(
        private Container $container,
        private array $packageConfig,
        private ?LoggerInterface $logger = null,
    ) {
        $this->logger = $logger ?? new NullLogger();
    }

    public function name(): string
    {
        return 'rabbitmq';
    }

    public function connect(array $config): Queue
    {
        $vendor = new VendorConnector($this->container['events']);
        $vendorQueue = $vendor->connect($config);

        // Re-wrap as our RabbitQueue. We can't trivially "cast" so we construct
        // a new instance using the vendor queue's exposed properties. The
        // RabbitQueue constructor mirrors the vendor's.
        return new RabbitQueue(
            connection: $vendorQueue->getConnection(),
            connectionName: $vendorQueue->getConnectionName(),
            queueName: $config['queue'] ?? 'default',
            options: $config['options'] ?? [],
            container: $this->container,
            packageConfig: $this->packageConfig,
        );
    }

    public function workload(array $queues): array
    {
        $connectionName = $this->packageConfig['transports']['rabbitmq']['workload_connection'] ?? 'rabbitmq';
        $config = config("queue.connections.{$connectionName}");

        $rows = [];

        if (! $config || empty($config['hosts'])) {
            // No connection configured; everything reports zero.
            foreach ($queues as $q) {
                $rows[] = ['name' => $q, 'length' => 0, 'processes' => 0, 'wait' => 0, 'split_queues' => null];
            }
            return $rows;
        }

        try {
            $host = $config['hosts'][0];
            $conn = new AMQPStreamConnection(
                $host['host'], $host['port'], $host['user'], $host['password'], $host['vhost']
            );
            $channel = $conn->channel();

            foreach ($queues as $queueName) {
                try {
                    [, $messageCount, ] = $channel->queue_declare($queueName, true);
                    $rows[] = [
                        'name' => $queueName,
                        'length' => (int) $messageCount,
                        'processes' => 0,
                        'wait' => 0,
                        'split_queues' => null,
                    ];
                } catch (Throwable $e) {
                    $this->logger->warning("Sunset RabbitMQ workload: queue '{$queueName}' depth failed: " . $e->getMessage());
                    $rows[] = ['name' => $queueName, 'length' => 0, 'processes' => 0, 'wait' => 0, 'split_queues' => null];
                }
            }

            $channel->close();
            $conn->close();
        } catch (Throwable $e) {
            $this->logger->warning('Sunset RabbitMQ workload: connection failed: ' . $e->getMessage());
            foreach ($queues as $queueName) {
                $rows[] = ['name' => $queueName, 'length' => 0, 'processes' => 0, 'wait' => 0, 'split_queues' => null];
            }
        }

        return $rows;
    }
}
```

### Step 6.1: Implement the queue

Create `src/Transports/Rabbit/RabbitQueue.php`:

```php
<?php

namespace Admnio\Sunset\Transports\Rabbit;

use Admnio\Sunset\Delay\DelayedJobStore;
use Admnio\Sunset\Events\JobQueued;
use Admnio\Sunset\Events\JobQueueing;
use Admnio\Sunset\Events\JobReserved;
use Admnio\Sunset\JobPayload;
use DateInterval;
use DateTimeInterface;
use Illuminate\Contracts\Container\Container;
use Illuminate\Support\Str;
use VladimirYuldashev\LaravelQueueRabbitMQ\Queue\RabbitMQQueue as VendorQueue;

class RabbitQueue extends VendorQueue
{
    private mixed $lastPushed = null;

    public function __construct(
        $connection,
        string $connectionName,
        string $queueName,
        array $options,
        private Container $container,
        private array $packageConfig,
    ) {
        parent::__construct($connection, $connectionName, $queueName, $options);
    }

    public function push($job, $data = '', $queue = null)
    {
        $this->lastPushed = $job;
        return parent::push($job, $data, $queue);
    }

    public function pushRaw($payload, $queue = null, array $options = [])
    {
        $prepared = (new JobPayload($payload))->prepare($this->lastPushed);
        $queueName = $queue ?? $this->getQueue($queue);
        $connection = $this->getConnectionName();

        event(new JobQueueing($connection, $queueName, $prepared));

        $result = parent::pushRaw($prepared->value, $queue, $options);

        event(new JobQueued($connection, $queueName, $prepared));

        return $prepared->id();
    }

    public function later($delay, $job, $data = '', $queue = null)
    {
        $this->lastPushed = $job;

        $queueName = $queue ?? $this->getQueue($queue);

        // Build the same payload we'd push via pushRaw, but route through
        // Sunset's DelayedJobStore instead of trying to use RabbitMQ-native
        // delay semantics. The sweep-delayed scheduled command picks it up
        // and dispatches into RabbitMQ at the right time.
        $payload = $this->createPayload($job, $queueName, $data);

        $availableAt = $this->availableAt($delay);

        $store = $this->container->make(DelayedJobStore::class);
        $store->store($this->getConnectionName(), $queueName, $payload, $availableAt);

        return Str::random(40);
    }

    public function pop($queue = null)
    {
        $job = parent::pop($queue);

        if ($job !== null) {
            $queueName = $queue ?? $this->getQueue($queue);
            $payload = new JobPayload($job->getRawBody());
            event(new JobReserved($this->getConnectionName(), $queueName, $payload));
        }

        return $job;
    }

    /**
     * Helper used by `availableAt`-style date math. Mirrors the calculation
     * Laravel's Queue base class does, scoped here so we don't depend on a
     * parent-class private method.
     */
    private function availableAt(int|DateInterval|DateTimeInterface $delay): int
    {
        if ($delay instanceof DateTimeInterface) {
            return $delay->getTimestamp();
        }
        if ($delay instanceof DateInterval) {
            return (new \DateTime())->add($delay)->getTimestamp();
        }
        return time() + $delay;
    }
}
```
