# Sunset v2.0.0 — Dashboard Redesign

> **For agentic workers:** Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Replace the current v1.x dashboard UI with the Linear/Vercel-style redesign approved in `mockups/dashboard-v3.html`. The mockup is the source of truth — port it to Vue/Inertia, keeping all existing data plumbing intact.

**Architecture:** No backend contract changes for v1.x routes. Two new pages (`class-detail`, `tag-detail`) with new controller methods. Frontend: full visual reskin + new shared components (CommandPalette, ToastContainer, FilterBar, RangeGroup, ChartCard, Histogram, BulkActionBar) + global keyboard shortcuts.

**Mockup:** `mockups/dashboard-v3.html` — every visual decision is locked in there.

### Key constraints

- **No breaking API changes.** All existing dashboard routes and prop shapes must remain identical. New props on Metrics/Monitoring pages must be added in a way that doesn't break existing consumers (additive only).
- **Mockup is source of truth.** Whenever a Vue component diverges from `mockups/dashboard-v3.html`, fix the Vue, not the mockup.
- **Use existing patterns where they fit.** Existing components like `ConfirmAction`, `Empty`, `MasterDetail`, `ExceptionTrace` should be reused; refresh their styling but keep their API.
- **Tests stay green at every commit.** Existing tests should continue to pass; new tests added for new endpoints/components.
